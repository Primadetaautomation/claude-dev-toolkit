### Content Accessibility Guidelines
```markdown
## Accessibility Content Standards

### Writing for Accessibility
- **Plain Language**: Simple, clear sentences and common vocabulary
- **Structure**: Logical heading hierarchy and scannable content
- **Alt Text**: Descriptive alternative text for images and graphics
- **Link Text**: Descriptive link text (not "click here" or "read more")

### Visual Content Accessibility
- **Color**: Don't rely solely on color to convey information
- **Contrast**: Ensure sufficient color contrast for text readability
- **Typography**: Readable fonts and appropriate sizing
- **Spacing**: Adequate white space and line spacing

### Technical Accessibility
- **Semantic HTML**: Proper use of headings, lists, and structural elements
- **Focus Management**: Keyboard navigation and focus indicators
- **Screen Reader**: Content optimized for assistive technologies
- **Forms**: Clear labels, error messages, and validation feedback

### Accessibility Checklist
- [ ] Content follows WCAG 2.1 AA guidelines
- [ ] All images have descriptive alt text
- [ ] Headings create logical document structure
- [ ] Links are descriptive and contextual
- [ ] Color contrast meets minimum requirements
- [ ] Content is keyboard accessible
- [ ] Forms have proper labels and error handling
- [ ] Videos have captions and transcripts
```
